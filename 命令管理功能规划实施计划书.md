# NetWeaverGo 命令管理功能规划实施计划书

## 一、需求概述

### 1.1 业务背景

当前 NetWeaverGo 项目的命令管理功能存在以下限制：

- 命令仅支持单一列表存储（config.txt），无法保存多套命令组
- 任务执行页面的命令编辑功能与执行流程耦合
- 无法灵活选择特定设备执行特定命令组

### 1.2 目标需求

1. **新增命令管理页面**：支持对命令进行编辑，保存成独立的命令组
2. **抽离命令编辑功能**：将任务执行页面的命令编辑器功能迁移到命令管理页面
3. **优化任务执行流程**：支持多方式筛选设备 → 选择命令组 → 下发执行

## 三、详细设计方案

### 3.1 数据模型设计

#### 3.1.1 命令组数据结构

```go
// CommandGroup 命令组定义
type CommandGroup struct {
    ID          string   `json:"id"`          // 唯一标识（UUID）
    Name        string   `json:"name"`        // 命令组名称
    Description string   `json:"description"` // 描述信息
    Commands    []string `json:"commands"`    // 命令列表
    CreatedAt   string   `json:"createdAt"`   // 创建时间
    UpdatedAt   string   `json:"updatedAt"`   // 更新时间
    Tags        []string `json:"tags"`        // 标签（用于分类）
}
```

#### 3.1.2 存储方案

采用 JSON 文件存储命令组列表，便于读写和维护：

```
commands/
├── groups.json              # 命令组索引文件
└── legacy_config.txt        # 保留原有格式兼容
```

**groups.json 结构**：

```json
{
  "groups": [
    {
      "id": "uuid-xxxx",
      "name": "基础信息采集",
      "description": "采集交换机基础配置信息",
      "commands": [
        "display version",
        "display device",
        "display interface brief"
      ],
      "createdAt": "2024-01-01T10:00:00",
      "updatedAt": "2024-01-01T10:00:00",
      "tags": ["信息采集", "基础"]
    }
  ]
}
```

## 五、详细实现规格

### 5.1 设备筛选方式

任务执行页面支持以下设备筛选方式：

| 筛选方式 | 说明               | 实现方式                       |
| -------- | ------------------ | ------------------------------ |
| 全选     | 选择所有设备       | `ListDevices()`                |
| 按分组   | 选择指定分组的设备 | 分组下拉列表                   |
| 按标签   | 选择指定标签的设备 | 标签下拉列表                   |
| 按协议   | 选择指定协议的设备 | 协议下拉列表 (SSH/SNMP/Telnet) |
| 手动选择 | 弹出设备列表多选   | 复用 Devices.vue 选择逻辑      |
| 搜索筛选 | 按关键词搜索后选择 | 模糊搜索 + 批量选择            |

### 5.2 命令组操作流程

```
创建命令组:
用户点击「新建」→ 弹出编辑弹窗 → 填写名称/描述/标签 → 编辑命令 → 保存

编辑命令组:
点击「编辑」→ 弹出编辑弹窗 → 修改内容 → 保存

删除命令组:
点击「删除」→ 确认弹窗 → 确认删除

复制命令组:
点击「复制」→ 自动创建副本（名称后缀 "-副本"）

导入/导出:
支持 JSON 格式导入导出，便于备份和分享
```

### 5.3 数据迁移策略

为确保兼容性，保留原有 `config.txt` 文件：

- 首次启动时，检测是否存在 `groups.json`
- 若不存在，自动将 `config.txt` 内容导入为默认命令组
- 命令组名称：`默认命令组`
- 标签：`系统默认`

## 七、风险与应对

| 风险          | 影响             | 应对措施                          |
| ------------- | ---------------- | --------------------------------- |
| 旧数据兼容    | 用户现有命令丢失 | 自动迁移 config.txt 到默认命令组  |
| JSON 文件损坏 | 数据丢失         | 定期备份 + 错误恢复提示           |
| 大量设备选择  | 性能问题         | 分页展示 + 虚拟滚动               |
| 命令组误删    | 操作不可逆       | 增加确认弹窗 + 软删除机制（可选） |
