## 模块功能概述

网络计算模块提供 IPv4/IPv6 地址计算功能，包括：
- IP 地址基本信息展示（网络地址、广播地址、子网掩码等）
- 子网划分计算（按主机数或子网数）
- IPv6 地址包含关系检查
- 子网划分结果 CSV 导出

## 审计发现

### 1. 安全性审计

#### 1.1 输入验证 ✅ 良好

**后端验证机制完善：**

- [`parseIPv4Addr()`](internal/ui/network_calc_service.go:531) 使用 Go 标准库 `netip.ParseAddr()` 进行严格验证
- [`parseIPv6Addr()`](internal/ui/network_calc_service.go:539) 正确处理 IPv6 地址及 zone 标识符（`%` 后缀）
- [`parseStrictInt()`](internal/ui/network_calc_service.go:559) 实现严格整数解析，拒绝前导零等异常格式
- [`parseMaskOrWildcard()`](internal/ui/network_calc_service.go:420) 支持三种掩码格式（CIDR、子网掩码、反掩码）并正确验证

**前端输入过滤：**

```typescript
// IPv4Calc.vue:108-114
watch(ipStr, (newVal) => {
  if (!newVal) return;
  const sanitized = newVal.replace(/[^0-9.]/g, "");
  if (sanitized !== newVal) {
    ipStr.value = sanitized;
  }
});
```

前端对 IP 地址输入进行字符过滤，仅允许数字和点号，提供良好的用户体验。

#### 1.2 整数溢出处理 ⚠️ 需关注

[`requiredHostBits()`](internal/ui/network_calc_service.go:588) 函数存在溢出检测：

```go
func requiredHostBits(hosts uint64) int {
    needed := hosts + 2  // 加上网络地址和广播地址
    if needed < hosts {  // 溢出检测
        return 32
    }
    // ...
}
```

**问题分析：** 当 `hosts` 接近 `uint64` 最大值时，`hosts + 2` 会溢出回绕。当前代码检测到溢出后返回 32（最大主机位数），这是合理的防御性处理，但可能导致用户困惑（无法理解为何需要 /32 网络）。

**建议：** 在上层调用处添加输入范围检查，提前拒绝明显不合理的请求（如超过 2^32-3 的主机数）。

#### 1.3 CSV 导出文件名安全 ✅ 良好

[`ExportIPv4SubnetsCSV()`](internal/ui/network_calc_service.go:229) 构建文件名时使用网络地址：

```go
baseNet := calc.Subnets[0].Network
if baseNet == "" {
    baseNet = "未知网络"
}
fileName := strings.ReplaceAll(baseNet, ".", "_") + "_子网划分明细.csv"
```

文件名仅进行简单的点号替换，但由于网络地址格式固定（如 `192.168.1.0`），不存在路径遍历风险。

### 2. 并发安全审计 ✅ 无问题

该模块为**无状态计算服务**：
- 所有计算函数都是纯函数，不依赖共享状态
- 无全局变量修改
- 无需并发控制

### 3. 性能审计 ✅ 良好

#### 3.1 子网数量限制

后端设置了合理的保护限制：

```go
const (
    maxIPv4SubnetsToGenerate = 256    // 默认最大生成数
    absoluteIPv4SubnetsLimit = 65535  // 强制显示上限
    maxIPv6SubnetsToGenerate = 256    // IPv6 默认上限
    maxIPv6SubnetDiff        = 16     // IPv6 单次下拨位数上限
)
```

#### 3.2 前端性能保护

前端 [`IPv4Calc.vue`](frontend/src/components/network/IPv4Calc.vue:41-55) 使用 ticket 机制防止过期响应覆盖：

```typescript
let calcTicket = 0;
const calculateFromBackend = async () => {
  const ticket = ++calcTicket;
  // ...
  if (ticket !== calcTicket) {
    return;  // 忽略过期响应
  }
  // ...
};
```

#### 3.3 潜在性能问题

[`CalculateIPv4()`](internal/ui/network_calc_service.go:97) 在生成大量子网时使用循环追加：

```go
for i := uint64(0); i < limit; i++ {
    result.Subnets = append(result.Subnets, IPv4SubnetItem{...})
}
```

当 `limit` 达到 65535 时，slice 会经历多次扩容。建议预分配容量：

```go
result.Subnets = make([]IPv4SubnetItem, 0, limit)
```

### 4. 逻辑漏洞审计 ✅ 良好

#### 4.1 边界条件处理

[`CalculateIPv4()`](internal/ui/network_calc_service.go:97) 正确处理 /31 和 /32 特殊网络：

```go
if targetCidr < 31 {
    usableHosts = (uint64(1) << uint(32-targetCidr)) - 2
    firstUsable = uint32ToIPv4(uint32(networkLong + 1))
    lastUsable = uint32ToIPv4(uint32(broadcastLong - 1))
} else if targetCidr == 31 {
    usableHosts = 2
    firstUsable = uint32ToIPv4(uint32(networkLong))
    lastUsable = uint32ToIPv4(uint32(broadcastLong))
} else {  // /32
    usableHosts = 1
    // ...
}
```

符合 RFC 3021 对 /31 网络的特殊处理要求。

#### 4.2 IPv6 类型判断

[`getIPv6Type()`](internal/ui/network_calc_service.go:702) 实现了完整的 IPv6 地址类型识别，包括：
- 未指定地址 (::)
- 环回地址 (::1)
- 组播地址 (ff00::/8)
- 链路本地地址 (fe80::/10)
- 站点本地地址 (fec0::/10) - 已废弃标记
- 唯一本地地址 (fc00::/7)
- 全球单播地址 (2000::/3)

3. **用户体验**：当溢出检测触发时，返回更明确的错误信息
