# NetWeaverGo 状态机运行逻辑检查计划

## 检查目标

根据《状态机运行逻辑检查指导书》，对项目中4个核心状态机组件进行连续三次检查：

1. **EngineStateManager** - 引擎状态管理器
2. **SessionReducer** - 会话状态机
3. **GlobalEngineState** - 全局状态管理器
4. **SuspendManager** - 挂起会话管理器

---

## 检查轮次安排

### 第一次检查：基础状态转移测试

**目标**：验证状态机的基本状态流转是否正常

| 组件               | 检查项         | 测试命令                                                                                  |
| ------------------ | -------------- | ----------------------------------------------------------------------------------------- |
| EngineStateManager | 正常启动路径   | `go test ./internal/engine/... -run TestEngineStateTransitionsWithoutPaused -v`           |
| EngineStateManager | 状态字符串     | `go test ./internal/engine/... -run TestEngineStateStringWithoutPaused -v`                |
| SessionReducer     | 初始状态       | `go test ./internal/executor/... -run TestReducerInitialState -v`                         |
| SessionReducer     | 终态不处理事件 | `go test ./internal/executor/... -run TestReducerTerminalState -v`                        |
| SessionReducer     | 初始化流程     | `go test ./internal/executor/... -run TestReducerInitPromptToWarmup -v`                   |
| SessionReducer     | 预热到就绪     | `go test ./internal/executor/... -run TestReducerWarmupToReady -v`                        |
| SessionReducer     | 发送命令       | `go test ./internal/executor/... -run TestReducerSendCommand -v`                          |
| SuspendManager     | 并发安全       | `go test ./internal/ui/... -run TestSuspendManager_ConcurrentFinishAndResolve_NoPanic -v` |
| SuspendManager     | 结束后响应忽略 | `go test ./internal/ui/... -run TestSuspendManager_ResolveAfterFinish_IsIgnored -v`       |

**预期输出**：所有测试通过 (PASS)

---

### 第二次检查：边界条件和异常路径测试

**目标**：验证状态机在异常场景和边界条件下的行为

| 组件               | 检查项        | 测试命令                                                            |
| ------------------ | ------------- | ------------------------------------------------------------------- |
| EngineStateManager | 非法状态转移  | 编写并运行非法转移测试                                              |
| EngineStateManager | 终态不可转移  | 验证Closed状态转移限制                                              |
| SessionReducer     | 超时路径      | `go test ./internal/executor/... -run TestReducerCommandTimeout -v` |
| SessionReducer     | 错误匹配      | `go test ./internal/executor/... -run TestReducerErrorMatched -v`   |
| SessionReducer     | 分页处理      | `go test ./internal/executor/... -run TestReducerPagination -v`     |
| SessionReducer     | 流关闭        | `go test ./internal/executor/... -run TestReducerStreamClosed -v`   |
| SessionReducer     | 用户继续/中止 | `go test ./internal/executor/... -run TestReducerUserActions -v`    |
| SuspendManager     | 超时机制      | 验证5分钟超时逻辑                                                   |
| SuspendManager     | 重复响应      | 验证resolved标记防止重复                                            |

**预期输出**：所有测试通过 (PASS)，非法操作返回错误

---

### 第三次检查：集成测试和竞态条件检查

**目标**：验证跨组件交互和并发安全性

| 检查类型 | 检查项       | 测试命令                                                                         |
| -------- | ------------ | -------------------------------------------------------------------------------- |
| 全部测试 | 状态机相关   | `go test ./internal/engine/... ./internal/executor/... ./internal/ui/... -v`     |
| 竞态检查 | 数据竞争检测 | `go test -race ./internal/engine/... ./internal/executor/... ./internal/ui/...`  |
| 覆盖率   | 代码覆盖     | `go test -cover ./internal/engine/... ./internal/executor/... ./internal/ui/...` |
| 集成测试 | 跨组件       | 运行集成测试套件                                                                 |

**预期输出**：

- 所有测试通过
- 无数据竞争 (race detected: 0)
- 代码覆盖率 > 80%

```

Running                  → Ready, AwaitPagerContinueAck, Suspended, Failed
```

## 问题严重程度分级

| 级别 | 说明                 | 处理方式           |
| ---- | -------------------- | ------------------ |
| 严重 | 状态机卡死/死锁/崩溃 | 立即修复，阻塞发布 |
| 高   | 状态转移异常/不可达  | 必须修复           |
| 中   | 并发安全问题         | 建议修复           |
| 低   | 代码优化/日志改进    | 酌情处理           |

## 时间安排

| 轮次   | 预计时间 | 内容               |
| ------ | -------- | ------------------ |
| 第一次 | ~5分钟   | 基础状态转移测试   |
| 第二次 | ~10分钟  | 边界条件和异常路径 |
| 第三次 | ~15分钟  | 集成测试和竞态检查 |
| 汇总   | ~5分钟   | 生成最终报告       |

**总计约35分钟**

## 检查前准备

1. 确保Go环境正常：`go version`
2. 确保依赖已下载：`go mod tidy`
3. 确保项目可编译：`go build ./...`

---

## 备注

- 每次检查后需记录所有问题
- 三次检查应连续执行，避免代码变更
- 如发现问题，需记录问题描述、严重程度、建议修复方案
