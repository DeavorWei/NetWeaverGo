# NetWeaverGo 项目整改实施任务清单

> 基于 `docs/project_analysis_report.md` 与 `docs/project_analysis_solutions.md` 整理

> 更新时间：2026-03-20

---

## 1. 使用说明

本文按“优先级 + 依赖 + 验收标准”拆分整改任务。

优先级约定：

- P0：应优先处理，主要是安全暴露和低成本高收益项
- P1：建议第二批处理，主要是结构整理和可维护性提升
- P2：中等改造，通常需要联动多个模块
- P3：观察后再决定，或需要先拿到真实指标

状态约定：

- TODO：待开始
- READY：依赖已满足，可开始
- HOLD：有前置依赖，暂不开始
- DONE：已完成

## 4. 第一批任务

### T03. 收紧设备列表和查询接口的密码返回

- 优先级：P0
- 状态：READY
- 对应问题：S4、H5
- 目标：把密码查看能力收口到“单设备显式操作”，不允许列表和分页查询默认返回密码
- 范围：
  - `internal/ui/device_service.go`
  - `internal/ui/query_service.go`
  - `internal/models`
  - `frontend/src/services`
  - 相关设备页面与表格
- 依赖：无
- 输出：
  - 设备列表接口不再返回密码
  - 分页/搜索查询接口不再返回密码
  - 单设备查看/编辑接口保留按需返回能力
- 验收标准：
  - 设备列表、分页查询、搜索结果的响应中不带密码
  - 编辑单台设备时仍可按产品要求查看已有密码

### T04. 调整前端“查看已有密码”交互边界

- 优先级：P0
- 状态：HOLD
- 对应问题：S4
- 目标：允许查看已有密码，但只在单设备显式交互中获取和展示
- 范围：
  - 设备编辑弹窗或详情页面
  - 设备相关 store 和本地表单状态
- 依赖：T03
- 输出：
  - 只在编辑单台设备时请求密码
  - 密码不进入全局持久状态
  - 关闭编辑界面后清空临时密码状态
- 验收标准：
  - 进入设备列表不会批量拉取密码
  - 只有单设备编辑流程才会获取密码
  - 关闭编辑窗口后前端内存中不再保留该密码

### T05. 路由懒加载改造

- 优先级：P0
- 状态：READY
- 对应问题：A3、P4
- 目标：降低前端首屏加载成本
- 范围：
  - `frontend/src/router/index.ts`
  - 首次进入的高频页面 loading 表现
- 依赖：无
- 输出：
  - 非首屏页面改为动态导入
  - 为懒加载页面补 loading 或骨架屏
- 验收标准：
  - 路由组件不再全部静态导入
  - 首屏资源体积下降
  - 首次切换子页面时不会出现明显空白

### T06. 并发数配置统一收口

- 优先级：P0
- 状态：READY
- 对应问题：P3
- 目标：移除分散的硬编码并发配置
- 范围：
  - `internal/config/constants.go`
  - `internal/engine`
  - 发现、备份等并发相关模块
- 依赖：无
- 输出：
  - 统一并发数解析入口
  - 默认值、最大值、用户值的优先级定义清晰
- 验收标准：
  - 不再出现多个模块各自维护冲突的默认并发数
  - 用户设置与运行时实际并发行为一致

## 5. 第二批任务

### T07. 统一密码合并规则

- 优先级：P1
- 状态：READY
- 对应问题：R1
- 目标：单设备更新与批量更新使用同一套密码处理语义
- 范围：
  - `internal/config/config.go`
  - 设备新增/编辑/批量编辑相关接口
- 依赖：T03、T04
- 输出：
  - 明确“空值不修改”的统一规则
  - 返回“密码是否变化”的统一结果
- 验收标准：
  - 单设备更新和批量更新行为一致
  - 不会出现某个入口清空密码、另一个入口保留密码的分叉

### T08. Discovery 数据结构去重

- 优先级：P1
- 状态：READY
- 对应问题：R2
- 目标：消除重复匿名结构体，降低维护成本
- 范围：
  - `internal/discovery/runner.go`
  - 如有必要，补充 discovery 模块内部 DTO
- 依赖：无
- 输出：
  - 重复查询结果结构体抽成具名类型
- 验收标准：
  - 同类字段结构不再重复定义两次
  - 后续字段变更只需改一处

### T09. SSH 认证方法构造收口

- 优先级：P1
- 状态：READY
- 对应问题：R3
- 目标：统一密码认证和键盘交互认证构造逻辑
- 范围：
  - `internal/sshutil/client.go`
- 依赖：无
- 输出：
  - SSH 认证方法统一从一个入口生成
  - 认证方法构造与主机密钥策略分离
- 验收标准：
  - 同类 SSH 认证逻辑不再出现多份实现
  - 修改认证策略时只需改一处

### T10. 抽离设备资产 Repository/Store

- 优先级：P1
- 状态：READY
- 对应问题：A1
- 目标：降低 UI Service 与 `config` 包耦合
- 范围：
  - `internal/config`
  - `internal/ui`
  - 新增 `repository` 或 `store` 层
- 依赖：T07
- 输出：
  - 设备资产读写逻辑迁出 `config`
  - UI Service 面向接口，而不是直接调 `config.LoadXXX`
- 验收标准：
  - 设备相关 CRUD 不再由 `config` 包直接承载业务读写职责
  - UI Service 能通过接口替换 mock 实现

### T11. 前端状态订阅拆分

- 优先级：P1
- 状态：READY
- 对应问题：P6
- 目标：降低大快照对象带来的无效响应和渲染
- 范围：
  - `frontend/src/stores/engineStore.ts`
  - 执行状态相关页面
- 依赖：无
- 输出：
  - 高低频状态拆分
  - 深层对象响应范围收缩
- 验收标准：
  - 进度刷新时无关页面不发生明显重渲染
  - 大对象更新不会导致整页状态频繁抖动

### T12. 前端 API 类型来源统一

- 优先级：P1
- 状态：READY
- 对应问题：R5
- 目标：以 bindings 作为唯一后端 DTO 来源
- 范围：
  - `frontend/src/services/api.ts`
  - `frontend/src/bindings`
  - 使用到手写后端类型的前端模块
- 依赖：无
- 输出：
  - `api.ts` 保留聚合职责，不再重复定义后端 DTO
  - 手写类型仅保留视图态和表单态
- 验收标准：
  - 后端 DTO 类型来源唯一
  - bindings 更新后，前端类型漂移问题可被编译阶段发现

## 6. 第三批任务

### T13. Windows DPAPI 凭据保护改造

- 优先级：P2
- 状态：HOLD
- 对应问题：S1
- 目标：在 Windows 上用 DPAPI 替代当前明文密码存储
- 范围：
  - 凭据存储接口设计
  - 数据库凭据字段
  - 设备读取、编辑、执行连接等调用链
- 依赖：T03、T04、T07
- 输出：
  - 统一凭据存储接口
  - 数据库存储改为 DPAPI 保护后的密文
  - 查看密码与 SSH 连接走受控解密入口
- 验收标准：
  - 数据库不再直接存储明文密码
  - 设备执行和单设备查看密码仍正常工作
  - 不依赖恢复旧的 `.credential_key` 方案

### T14. SSH 主机密钥策略治理

- 优先级：P2
- 状态：READY
- 对应问题：S2
- 目标：保留 `insecure` 兼容能力，但保证显式风险提示和审计
- 范围：
  - `internal/sshutil`
  - 设置服务与前端设置页
  - 审计日志相关模块
- 依赖：无
- 输出：
  - 默认策略固定为 `strict`
  - `insecure` 放到高级区并显式确认
  - `accept_new` 保存首次指纹并监测变化
  - 连接时写入策略审计日志
- 验收标准：
  - 新用户默认不会落到 `insecure`
  - 使用 `insecure` 或 `accept_new` 时有可见提示和日志记录

### T15. Engine 生命周期治理

- 优先级：P2
- 状态：HOLD
- 对应问题：A2
- 目标：收口状态流转和关闭流程，降低并发维护复杂度
- 范围：
  - `internal/engine`
  - 与前端状态同步的服务层
- 依赖：T06
- 输出：
  - 状态流转表
  - 统一生命周期入口
  - 状态副作用收口
- 验收标准：
  - 状态转换路径清晰
  - 关闭、暂停、恢复流程不会依赖分散的隐式约定

### T16. 事件总线背压与分级

- 优先级：P2
- 状态：HOLD
- 对应问题：P2
- 目标：防止高频事件堆积导致 UI 延迟和内存上涨
- 范围：
  - `internal/engine`
  - 事件总线与 tracker 相关模块
- 依赖：T15
- 输出：
  - 关键事件与进度事件分级
  - 高频事件合并或限流策略
  - 队列长度和丢弃指标
- 验收标准：
  - 进度高频刷新的场景下内存和 UI 延迟保持可控
  - 关键事件不丢失

  - 存储根目录
  - 设备字段输入校验
- 依赖：无
- 输出：
  - 统一路径校验入口
  - 统一字段校验规则
  - 批量操作的逐条错误反馈
- 验收标准：
  - 高风险路径不会被 `..` 或绝对路径绕过
  - 批量导入和编辑时错误信息可定位
