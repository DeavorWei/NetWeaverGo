## 审计发现

### 1. 安全性审计

#### 1.1 输入验证 ⚠️ 需关注

**前端输入验证：**

前端组件使用 Element Plus 的表单组件，部分输入缺少严格验证：

- **存储路径输入** ([`Settings.vue:76`](frontend/src/views/Settings.vue:76))：仅使用 `el-input`，未限制路径格式
- **超时参数输入** ([`Settings.vue:54-61`](frontend/src/views/Settings.vue:54-61))：接受任意字符串格式（如 "10s"），前端未验证格式正确性

**后端验证机制：**

[`SaveSettings()`](internal/config/settings.go:108) 对存储路径有验证：

```go
normalizedStorageRoot := NormalizeStorageRoot(settings.StorageRoot)
if err := ValidateStorageRootWritable(normalizedStorageRoot); err != nil {
    return err
}
```

但**超时参数缺少格式验证**，仅依赖字符串传递，可能导致解析失败。

#### 1.2 敏感数据处理 ⚠️ 需关注

**SSH 算法配置暴露：**

[`GetSSHAlgorithmOptions()`](internal/ui/settings_service.go:124) 返回完整的 SSH 算法列表，包括标记为 "insecure" 的不安全算法：

### 2. 并发安全审计 ⚠️ 需关注

#### 2.1 运行时配置并发控制 ✅ 良好

[`RuntimeConfigManager`](internal/config/runtime_config.go:404) 使用读写锁保护配置：

```go
type RuntimeConfigManager struct {
    db     *gorm.DB
    config RuntimeConfig
    mu     sync.RWMutex  // 读写锁
}
```

更新配置时正确加锁 ([`UpdateConfig()`](internal/config/runtime_config.go:522-535))：

```go
func (m *RuntimeConfigManager) UpdateConfig(config RuntimeConfig) error {
    // 保存到数据库
    if err := SaveRuntimeConfig(m.db, config); err != nil {
        return err
    }

    // 更新内存中的配置
    m.mu.Lock()
    m.config = config
    m.mu.Unlock()

    return nil
}
```

读取时使用读锁 ([`GetConfig()`](internal/config/runtime_config.go:515))：

```go
func (m *RuntimeConfigManager) GetConfig() RuntimeConfig {
    m.mu.RLock()
    defer m.mu.RUnlock()
    return m.config
}
```

#### 2.2 全局设置并发问题 ⚠️ 潜在风险

[`SaveSettings()`](internal/config/settings.go:108) 和 [`LoadSettings()`](internal/config/settings.go:52) 未使用锁机制：

- 多个前端用户同时保存设置可能导致竞态条件
- 数据库操作与内存状态更新之间存在时间窗口

**建议：** 为全局设置添加互斥锁保护，或使用数据库事务保证一致性。

#### 2.3 单例初始化安全 ✅ 良好

[`InitRuntimeManager()`](internal/config/runtime_config.go:416) 使用 `sync.Once` 保证单例安全初始化：

## 审计结论

### 风险评估

| 类别 | 评级 | 说明 |
|------|------|------|
| 安全性 | 中等风险 | SSH 不安全算法暴露、超时参数格式验证缺失 |
| 并发安全 | 低风险 | 运行时配置有锁保护，全局设置需加强 |
| 性能 | 低风险 | 配置加载效率良好 |
| 逻辑完整性 | 低风险 | 边界条件处理完善 |

### 改进建议

#### 高优先级

1. **超时参数格式验证**：在后端 [`SaveSettings()`](internal/config/settings.go:108) 入口添加超时字符串格式验证，拒绝非法格式（如非数字、负值）

2. **全局设置并发保护**：为 [`LoadSettings()`](internal/config/settings.go:52) 和 [`SaveSettings()`](internal/config/settings.go:108) 添加互斥锁，防止并发竞态

#### 中优先级

3. **SSH 算法安全警告增强**：在选择不安全算法时，要求用户二次确认或签署安全免责声明

4. **前端输入校验**：为存储路径添加基本格式验证（如禁止绝对路径注入攻击字符）

#### 低优先级

5. **配置变更审计日志**：记录配置变更历史，支持回滚操作

### 审计总结

系统设置模块整体架构合理，运行时配置并发控制完善，但存在以下需关注的问题：

1. **安全性**：SSH 不安全算法选项暴露可能诱导用户错误选择；超时参数缺少格式验证可能导致运行时错误
2. **并发安全**：全局设置缺少锁保护，存在潜在竞态条件风险

建议优先实施超时参数验证和全局设置并发保护，以提升模块安全性和稳定性。
