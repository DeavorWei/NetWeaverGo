## 1. 模块概述

路由追踪模块提供网络路径探测功能，支持单次探测和持续探测两种模式，具备实时跳数更新、GeoIP地理位置解析、路径变化检测等特性。

#### 原子操作使用

[`runRound()`](internal/icmp/tracert_engine.go:73) 使用原子操作：

```go
var reachedDest int32 = 0
var reachedTTL int32 = int32(maxHops + 1)

// 使用CAS更新最小TTL
if atomic.CompareAndSwapInt32(&reachedTTL, oldReachedTTL, int32(ttlVal)) {
```

**评估**: 正确使用原子操作避免竞争。

### 3.2 取消机制 ✅ 优秀

#### Context取消传播

[`runRound()`](internal/icmp/tracert_engine.go:73) 在多处检查取消：

```go
select {
case <-ctx.Done():
    logger.Debug("Tracert", targetIP, "检测到取消信号，停止启动新探测")
    // 发送 cancelled 结果
    for t := ttl; t <= maxHops; t++ {
        resultChan <- TracertHopResult{
            TTL:    t,
            Status: "cancelled",
        }
    }
    break
default:
}
```

#### 持续探测取消

[`runContinuous()`](internal/ui/tracert_service.go:285) 正确处理取消：

```go
select {
case <-ctx.Done():
    logger.Info("TracertService", target, "持续探测被停止")
    return
default:
}
```

### 3.3 并发风险点

#### ⚠️ 中风险：全量TTL并发探测

[`runRound()`](internal/icmp/tracert_engine.go:73) 启动maxHops个并发goroutine：

```go
for ttl := 1; ttl <= maxHops; ttl++ {
    wg.Add(1)
    go func(ttlVal int) {
        defer wg.Done()
        // 探测当前 TTL
        hopResult := e.probeHop(ctx, ip, ttlVal, opts)
        resultChan <- hopResult
    }(ttl)
}
```

**问题**: 当MaxHops=255时，会同时启动255个goroutine。

**评估**: 虽然数量可控，但建议添加并发限制。

```go
func isValidDomain(domain string) bool {
    // 简单域名格式验证
    if len(domain) > 253 {
        return false
    }
    domainRegex := regexp.MustCompile(`^([a-zA-Z0-9]([a-zA-Z0-9\-]{0,61}[a-zA-Z0-9])?\.)+[a-zA-Z]{2,}$`)
    return domainRegex.MatchString(domain)
```
