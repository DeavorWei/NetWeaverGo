## 二、迁移功能详细分析

### 2.1 IP/子网计算器（优先级：高）

**源文件**:

- `ToolsWeb/src/views/NetworkCalc.vue`
- `ToolsWeb/src/components/network/IPv4Calc.vue`
- `ToolsWeb/src/components/network/IPv6Calc.vue`

**功能清单**:

| 子功能               | 描述                      | 迁移复杂度 |
| -------------------- | ------------------------- | ---------- |
| IPv4 基础计算        | 网络/广播地址、可用主机数 | 低         |
| 多格式掩码支持       | CIDR、十进制掩码、反掩码  | 低         |
| 子网划分（按主机数） | 自动计算最优子网划分      | 中         |
| 子网划分（按子网数） | 指定子网数量划分          | 中         |
| CSV 导出             | 子网划分结果导出          | 低         |
| IPv6 地址计算        | 展开/压缩、类型识别       | 中         |
| IPv6 子网划分        | 前缀长度计算              | 中         |

**技术要点**:

- 纯前端计算，无需后端支持
- IPv6 使用 `BigInt` 处理 128 位地址
- 性能保护机制：默认展示 256 条，最大 65535 条

### 2.2 协议与端口速查（优先级：中）

**源文件**: `ToolsWeb/src/views/ProtocolPortRef.vue`

**数据资产**:

- 常用端口: 22 条记录 (SSH, FTP, DNS, HTTP, BGP, SNMP 等)
- IP 协议号: 15 条记录 (ICMP, TCP, UDP, OSPF, GRE 等)

**迁移方案**: 将静态数据迁移为 Go 后端管理，通过 Wails 绑定暴露

### 2.3 配置生成器（优先级：高）

**源文件**: `ToolsWeb/src/views/ConfigForge.vue`

**功能清单**:

| 子功能       | 描述                                 | 迁移复杂度 |
| ------------ | ------------------------------------ | ---------- |
| 模板变量替换 | `[A]`, `[B]` 占位符                  | 低         |
| 语法糖展开   | `1-10`, `vlan10-13`, `192.168.1.1-3` | 中         |
| 等差数列补全 | 智能推断变量序列                     | 中         |
| 三栏布局     | 模板/变量/预览                       | 低         |
| 文件导出     | 合并下载、分块下载                   | 中         |

**与 NetWeaverGo 的协同**:

- 可与 `config.txt` 命令集管理整合
- 生成的配置可直接用于任务执行

### 2.4 网络测速面板（优先级：低）

**源文件**:

- `ToolsWeb/src/views/NetworkPanel.vue`
- `ToolsWeb/src/workers/networkPanel.worker.ts`

**技术挑战**:

- Web Worker 在 Wails 桌面应用中的兼容性
- 需验证 Wails Webview 对 Web Worker 的支持程度

**建议**: 作为独立模块保留，后期评估迁移

## 五、技术适配方案

### 5.1 Tailwind CSS 版本适配

ToolsWeb 使用 Tailwind CSS 4.x，NetWeaverGo 使用 3.x。需处理：

| 差异项     | 处理方案                              |
| ---------- | ------------------------------------- |
| 新增工具类 | 检查 4.x 新特性，使用 3.x 等效写法    |
| 配置文件   | 保持 NetWeaverGo 现有 tailwind.config |
| 暗色模式   | 两者均使用 `dark:` 前缀，兼容         |

### 5.2 Wails 绑定适配

部分功能可考虑后端实现：

```go
// internal/tools/subnet/calculator.go (可选)
type SubnetCalculator struct{}

func (s *SubnetCalculator) CalculateIPv4(ip string, cidr int) *IPv4Result {
    // Go 实现，更高性能
}

// internal/ui/app.go
type AppService struct {
    // ...existing
    SubnetCalc *subnet.SubnetCalculator
}
```

### 5.3 文件系统集成

配置生成器的导出功能需适配 Wails 文件系统：

```typescript
// 使用 Wails Runtime API
import { SaveFile } from "@wailsio/runtime";

async function exportConfig(content: string, filename: string) {
  await SaveFile({
    DefaultFilename: filename,
    Data: content,
  });
}
```

## 七、风险评估与缓解措施

| 风险                | 影响           | 缓解措施                   |
| ------------------- | -------------- | -------------------------- |
| Tailwind 版本不兼容 | UI 样式异常    | 使用兼容写法，渐进升级     |
| Web Worker 兼容性   | 测速功能不可用 | 暂缓迁移，验证后再实施     |
| 文件导出权限        | 导出失败       | 使用 Wails 文件系统 API    |
| 大数据量性能        | 子网划分卡顿   | 保留性能保护机制，虚拟滚动 |
| UI 风格不统一       | 用户体验下降   | 统一设计规范，复用现有组件 |

```
优先级排序：
```
